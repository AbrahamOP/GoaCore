# GoaCloud — Notes pour Claude

## Workflow Git

- **Toujours pousser le code sur la branche `dev`** (jamais directement sur `main`).
- Si une branche de travail dédiée est demandée, créer la branche à partir de `dev` et merger / pusher ensuite vers `dev`.
- Ne pas créer de pull request sans demande explicite de l'utilisateur.
