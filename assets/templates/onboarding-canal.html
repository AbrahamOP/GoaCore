Canal & cloud de sauvegarde

GoaCore génère une clé ed25519, vous la déployez en root sur votre Proxmox via un script auditable, puis vous connectez votre cloud. GoaCore n'ouvre jamais de session SSH pour installer : c'est vous qui exécutez.

Mettre en place le canal

1
Générer la clé ed25519

GoaCore génère une paire de clés. La clé privée est chiffrée en base et ne quitte jamais le serveur ; seule la clé publique part dans le script. {{with .Channel}}{{if .HasInAppKey}}
Une clé existe déjà. La régénérer révoquera l'ancienne : vous devrez relancer le script d'installation sur le Proxmox.{{end}}{{end}}

2
Exécuter le script en root sur le Proxmox

Copiez-collez cette commande dans un shell root de votre Proxmox. Elle télécharge le script, vous le faites relire (less) puis l'exécute. Le script est idempotent et vérifie l'empreinte sha256 du helper embarqué. 

curl -k -fsSL {{.InstallerURL}} -o /tmp/goabackup-install.sh && less /tmp/goabackup-install.sh && sudo bash /tmp/goabackup-install.sh
Voir le script Voir le helper
3
Vérifier l'installation

Preuve de bout en bout : GoaCore interroge le canal (disk-free) via la commande forcée. Un succès confirme que la clé est acceptée et que le helper répond.

Sauvegarde cloud

4
Connexion cloud (off-site)

Le cloud se configure côté Proxmox avec rclone (GoaCore n'écrit jamais votre rclone.conf et reste en push-only). Une recette par backend :

S3 / Backblaze B2 (recommandé PME — sans navigateur)
rclone config   # n) nouveau remote → type s3/b2 → clés API → terminé
Google Drive (headless) (avancé)

OAuth Drive nécessite un navigateur : lancez rclone authorize "drive" sur votre poste, puis recollez le token dans rclone config sur le Proxmox. Alternative sans navigateur : compte de service (Service Account JSON) + Shared Drive.

Énumère les remotes rclone configurés sur votre Proxmox.
Prérequis réseau (hors script)
  • OPNsense doit autoriser SSH (port 22) depuis GoaCore vers votre Proxmox.
  • L'utilisateur de service créé est {{.ChannelUser}} (shell /bin/bash, sudo restreint au helper).
  • Pour l'off-site, un rclone.conf root préexistant est requis (étape 4).
